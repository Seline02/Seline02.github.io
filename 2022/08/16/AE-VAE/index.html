无监督学习

  • 打标注依赖于人
  • 大量存在没有label的数据

Auto-Encoder原理

  • unsupervised learning 也不是没有目标 auto-encoder目标是它自己

  • encoder和decoder也可以看成搭在一起才是一个神经网络

  • 中间的code是暂存的数据

  • 输入输出维度一样，保证能重建

  • 中间的neck可升维/降维 一般是降维 但仍保持语义相关性

  • 如果中间的neck降到两维，有一个好处是方便可视化，知道数据的分布

  • real-valued inputs：用MSE，就是element-wise地相减

  • binary inputs：用交叉熵。像素用0/1表示，每个像素可以看成一个label，输出是 0 ~ 1 的值

  • PCA是线性变换的，局限性大

  • PCA重建效果不好

Auto-Encoder变种

  • 变种：加噪声扰动 element-wise地加高斯噪声

  • 更加robust，防overfitting

  • $x$ -> $h$（hidden）-> $\bar{x}$

  • 右图是hidden层随训练epoch的分布变化，比如从 [1, 3] 随着 training 变到 [1, 10]

  • 想要人为地让它符合一个分布

  • 如果鉴别器认为这个分布不好，输出false；如果认为属于guassian分布的话，输出1

变分Auto-Encoder引入

  • 希望从 $z$ 中经过 $\phi$ 网络采样出 $x_i$ 的概率最大，其中 $z$ 的分布是由 $\theta$ 网络决定的

  • $z$是 $q_\theta$网络给定 $x_i$ 得来的

  • 要使 $z$ 再经过生成网络 $p_\theta$ 生成出 $x_i$ 的概率最大

  • 左边就是重建误差 也就是要让MSE最小

  • 右边使分布接近

  • Adversarial AE是通过discriminor使分布接近，这里是通过KL散度使分布接近

  • 固定蓝色的分布 调整红色的分布

  • 分布越相近 KL越小

  • $p$是预设的分布 $q$是实际的分布
  • 要使 $q$ 接近于 $p$，所以要计算 $KL(q||p)$ （图中顺序可能有问题）
  • 推导：https://zhuanlan.zhihu.com/p/345095899

Reparameterization trick

  • variational autoencoder中h得到的不是一个向量，而是一个分布。和auto encoder（determinsitic）做区别
  • auto encoder：$x$ -> $h$ -> $\bar{x}$ 是一个 deterministic 的过程；
  • variational autoencoder：$x$ -> $h\sim q(h)$ -> $\bar{x}$ ，用 $\mu,\sigma^2$ 来表达这个分布，从分布中采样 sample 出一个 $z$，再重建，是stocastic

  • sample的过程不可微分

  • $z$ 表示成 $\mu + \sigma \bigodot \epsilon$ ，其中 $\epsilon$ 是固定的

  • $z$ 对 $\mu,\sigma$ 都可导了

  • 对 $\epsilon$ 不可导， 但不需要更新 $\epsilon$

  • 如果没有 KL 那一块，产生的分布 variance 就会比较小。结果 $z$ 在蓝色那一小段少量变化时，$\bar{x}$ 就会急剧变化；$z$ 在边缘变化时，$\bar{x}$ 又不怎么变

变分自编码器VAE

  • $z$ 表示成 $\mu + \sigma \bigodot \epsilon$ 中间画绿色的四条线都可导了
  • 中间那两层不是全连接，根据 $\mu_1$ 和 $\sigma_1$ sample 出 $z_1$，根据 $\mu_2$ 和 $\sigma_2$ sample 出 $z_2$

  • Fashion Mnist
  • 右边是VAE重建出来的，比AE效果好

  • VAE 学习的是 $q(h)$
  • sample 不同的 $h$，送到后面的 decoder，得到 $\bar{x}$
  • 左图横向是 $h_0$ 改变，纵向是 $h_1$ 改变

  • auto encoder 不是生成模型 没有随机性在里面 是一一对应的

  • adversarial AE 加了一个鉴别器使得 h 接近预设的高斯分布；VAE 则通过把目标写进损失函数，使得达到预设的分布

  • variational AE 学的是分布 distribution 可以sample

  • VAE 比 GAN 更好 training

GAN

  • 对于Discriminator来说，从training set中sample的 $P_r(x)$（r表示real data）应该鉴别为1，自己生成的 $P_g(x)$应该鉴别为0

  • 绿色的自己生成的分布，黑色的是要学的Gaussian分布

  • 扩大：一种用padding；一种可以用deconvolution

  • 高维数据情况下，$P_G,P_{data}$不一定会overlap

  • 在不overlap的情况下，JS divergence的表现

  • 只要没有交叉，这两个分布的KL散度都是无穷大

  • KL和JS散度都有梯度弥散的情况
  • 所以GAN在刚开始train的时候会有不稳定的情况

  • 只要分布没有重叠，JS都一样，都是 $\log 2$，对generator的导数都是0

  • discriminator可以很好地区分，但是对generator导数为0，一开始很难优化

  • 分布一样后，鉴别器对每个样本都判断为1/2，达到纳什均衡点

  • P变成Q需要多少步

  • f学的是earth mover distance

  • WD解决了JS早期不能convergence的问题

  • 1-Lipschitz保证smooth
  • 通过做weight clipping来保证1-Lipchitz
  • 但是加了regularization后网络参数有限，表达能力有限

  • 写进损失函数，让导数靠近1。本来应该是-1到1，这里写成-1的平方是工程上的实现

  • $\hat{x}$是sample自 $x_r,x_g$之间的点，可以从 $x_r$ 中sample一个点，从 $x_g$ 中sample一个点，从这两个点之间插值出一个点。所有这样的点的导数做平均

https://blog.csdn.net/qq_41196612/article/details/109528221

中间的隐变量z（高斯分布）经过神经网络能拟合任何分布

以Q ( z ) 为概率密度的分布(可以看作是z的后验分布)中采样，在这里我们先假设分布是任意的去推导公式，最后再加条件简化推出来的优化目标。

Q(z)是人为指定的（proposal），要和真正的后验p(z|x)越接近越好，用Q(z|x)表示近似的后验分布，given x，表示输入x，通过网络训练这个后验分布（重参数化技巧）

https://blog.csdn.net/qq_41196612/article/details/109528221 推导的思路就是变分推断的思路

p(z)是先验，认为是高斯分布
