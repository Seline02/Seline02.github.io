Proximal Policy Optimization

发表于2022-08-21|更新于2022-11-29|深度学习强化学习
|阅读量:
  • 如果要 learn 的 agent 和与环境互动的 agent 是同一个的话，叫 on-policy，自己互动自己学；不是同一个的话，叫 off-policy

  • 想要用 θ′\theta^{'}θ′ collect 到的 data 去训练 θ\thetaθ ，意味着可以把 θ′\theta^{'}θ′ collect 到的 data 用很多次。可以执行 gradient ascent，update 参数很多次

  • 现在没法从 ppp 这个 distribution 中 sample data，只能从 qqq 中 sample data

  • 需要乘一个 weight p(x)q(x)\frac{p(x)}{q(x)}q(x)p(x)​ 修正这两个 distribution 的差异。但是不能 q(x)=0q(x)=0q(x)=0 时，p(x)p(x)p(x) 却不为 0，这样没有定义

  • 实际上 p,qp,qp,q 不能差太多。期望相等，方差不等
  • 理论上只要对 p,qp,qp,q sample 足够多次，得到的结果是一样的；但假设 sample 的次数不够多，因为他们的 variance 差距大，所以可能得到较大的差别
  • 图为当 p,qp,qp,q 相差较大时
  • 如果 sample 次数较少，ppp 只 sample 到左侧，qqq 只 sample 到右侧。结果会等式左侧为负，等式右边为正
  • θ′\theta^{'}θ′ 负责和环境做互动，示范给 θ\thetaθ 看。本来是从 pθp_{\theta}pθ​ 做 sample，现在从 pθ′p_{\theta^{'}}pθ′​ 做 sample
  • 和环境互动的是 θ′\theta^{'}θ′ 而不是 θ\thetaθ ，所以 sample 出来的东西和 θ\thetaθ 本身是没有关系的。所以可以让 θ′\theta^{'}θ′ 做互动，sample 一大堆 data 后，θ\thetaθ 可以 update 很多次
  • 现在应该把 Aθ(st,at)A^{\theta}(s_t,a_t)Aθ(st​,at​) 换成 Aθ′(st,at)A^{\theta^{'}}(s_t,a_t)Aθ′(st​,at​) 。AAA 这一项是想要估测说现在在某一个 state，采取某一个 action，接下来会得到的 accumulated reward 的值剪掉 baseline。现在 advantage 是根据 θ′\theta^{'}θ′ 估计出来的。这里假设这两项差不多
  • 会看到什么 state 往往和会采取什么 action 没有太大的关系，因此假设 pθ(st)p_{\theta}(s_t)pθ​(st​) 和 pθ′(st)p_{\theta^{'}}(s_t)pθ′​(st​) 差不多，而且这两项不好算。但是 pθ(at∣st),pθ′(at∣st)p_{\theta}(a_t|s_t),p_{\theta^{'}}(a_t|s_t)pθ​(at​∣st​),pθ′​(at​∣st​) 是好算的，把 sts_tst​ 带入网络，就会告诉我们每一个 ata_tat​ 的几率是多少
  • 整体的这一项是 gradient，我们从 gradient 去反推原来的 objective function Jθ′(θ)J^{\theta^{'}}(\theta)Jθ′(θ)。用的主要是 ∇f(x)=f(x)∇log⁡f(x)\nabla f(x)=f(x)\nabla\log f(x)∇f(x)=f(x)∇logf(x) 这个式子
  • 如果 pθp_{\theta}pθ​ 和 pθ′p_{\theta^{'}}pθ′​ 差太多，importance sampling 结果就会不好
  • 如何避免两者差太多，这个就是 PPO 在做的事情
  • 在 training 时多加一个 constraint。这个 constraint 是 θ,θ′\theta,\theta^{'}θ,θ′ 这两个 model 它们 output 的 action 的 KL divergence。注意这里不是把 θ,θ′\theta,\theta^{'}θ,θ′ 当作两个 distribution 算它们之间的距离
  • 考虑的不是参数上的距离，而是 action 上的距离，是因为很有可能对 actor 来说，参数的变化和 action 的变化，不一定完全一致
  • TRPO 是 PPO 的前身
  • 可以让 θ\thetaθ update 很多次
  • 动态调整 β\betaβ
  • clip 这一项的理解：第一项小于第二项的话，就 output 1−ϵ1-\epsilon1−ϵ，第一项如果大于第三项的话，就 output 1+ϵ1+\epsilon1+ϵ
  • 横轴是 pθ(at∣st)pθk(at∣st)\frac{p_{\theta}(a_t|s_t)}{p_{\theta^{k}}(a_t|s_t)}pθk​(at​∣st​)pθ​(at​∣st​)​，纵轴是 clip 这个 function 实际的输出
  • min 函数前面的那一项是绿色的那条线
  • 如果 AAA 大于 0，即 st,ats_t,a_tst​,at​ 的 pair 是好的，我们就想要增加这个 state/action pair 的几率，想要让 pθ(at∣st)p_{\theta}(a_t|s_t)pθ​(at​∣st​) 越大越好，但同时要限制它和 pθ′(at∣st)p_{\theta^{'}}(a_t|s_t)pθ′​(at​∣st​) 的比值不超过 1+ϵ1+\epsilon1+ϵ。只要比值超过，就没有 benefit 了；AAA 小于 0，同理
