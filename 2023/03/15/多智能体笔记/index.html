2.1 智能Agent

Agent和环境

  • Agent核心概念：自治性

  • Agent的定义：Agent是处在某个环境中的计算机系统，该系统有能力在这个环境中自主行动以实现其设计目标

  • Agent和对象的区别

    • Agent是自治的、灵活的、主动的
    • Agent是主动的、有利益最大化的目标；对象是被动的、没有利益最大化的目标

  • Agent和专家系统的区别

    • 经典专家系统是与现实分离的，与任何环境没有直接联系
    • 一般不能采取反应式、预动的行动
    • 一般没有社会行为能力，不能进行合作、协调和协商

  • 书上：

  • 对象 VS Agent

    • 自治：与对象相比Agent有更强的自治性，特别是它们自己决定是否接受其他Agent的请求执行一个动作
    • 灵活：Agent具有灵活的（反应的、预动的、社会的）行为能力，标准的对象模型根本没有这种行为能力
    • 主动：多Agent系统本质上是多线程的，其中每个Agent至少假设有一个控制线程

  • 相似处

    • 都封装了一些状态
    • 可以在这些状态下执行某些动作或者方法
    • 可以通过消息传递进行通信

  • 智能Agent应具有哪些能力

    • 反应性

      • 一个反应式的系统能与环境持续地交互，对环境发生的变化以及时的方式作出反应

    • 预动性

      • 通过主动发起动作，表现出目标引导的行为

    • 社会能力

      • 通过合作、协调、协商与其他Agent交互的能力
        • 合作：作为一个团队一起工作以达到一个共享的目标
        • 协调：如何管理Agent活动之间的内部依赖关系
        • 协商：Agent就共同关心的问题达成一致的能力

2.2 智能Agent的体系结构

智能Agent的抽象体系结构

  • Agent = 体系结构 + 程序
    • 程序：把感知信息映射到行动的函数

智能Agent的实现型体系结构

  • 知识型（演绎推理Agent）
    • 具有 表示Agent内部状态的部件
    • 内部状态：Agent所拥有的知识
    • 动作选择：一个基于其知识的逻辑演绎或定理证明过程
    • 各个部件：感知函数、状态转移函数、动作选择函数
  • 思维型（实用推理Agent）
    • 具有 表示Agent内部状态的部件
    • 内部状态：Agent所具有的思维状态
    • 动作选择：一个针对其内部思维状态的实用推理过程
      • 实用推理：慎思+目标手段推理
  • 反应型（反应式Agent）
    • 不具有 表示Agent内部状态的部件
    • 内部 定义了一系列的反应式规则
    • 动作选择：一个从情景到动作的映射
  • 混合型（混合式Agent）
    • 内部 集成了多个不同类型的实现型体系结构
    • 运行时 表现出多种形式的行为特征：自治性、反应性、预动性、社会性

2.3 演绎推理Agent

作为定理证明器的Agent

面向Agent的程序设计

  • 核心思想：用意识属性直接对Agent进行程序设计
  • Agent0：第一个实现面向Agent程序设计风格的语言
    • 一组能力集合（可以做的）
    • 初始信念集合
    • 初始承诺集合（将要做的）
    • 承诺规则集合（决定Agent如何行动的关键部分）
      • 一个消息条件
      • 一个思维状态条件
      • 一个动作

并发MetateM

  • 基于时序逻辑的多Agent编程语言
  • 时序逻辑
    • 命题时序逻辑=命题逻辑+事件的时间顺序模态
    • 一阶时序逻辑=一阶逻辑+事件的时间顺序模态
  • 并发MetateM系统中的Agent是并发执行的实体，可以通过消息广播互相通信
  • 并发MetateM Agent两个主要部件
    • 接口：定义Agent如何与环境（即其他Agent）进行交互
      • Agent的标识，环境命题集（接收…)，组件命题集（发送…)
      • stack(pop,push)[popped,full]
    • **计算引擎：**定义Agent如何采取行动
      • 每个时序逻辑公式描述了Agent的功能和行为
  • 程序规则：与过去有关的前件 ⇒\Rightarrow⇒ 与现在和将来有关的后件

2.4 实用推理Agent

  • 实用推理=慎思过程+目标手段推理

    • 慎思过程的结果是意图
    • 目标手段推理的结果是规划

  • 在实用推理中，用意图刻画思维状态

  • 信念/愿望/意图（BDI）模型

    • 信念修正函数
    • 选项生成函数（生成愿望集）
    • 过滤函数（生成意图集）

  • 盲目承诺Agent

    • 对手段和目标都盲目承诺

  • 仅对目标盲目承诺的Agent

    • sound(π,I,B)sound(\pi,I,B)sound(π,I,B) 代表在给定 BBB 时，π\piπ 是实现 III 的正确的规划
    • 对手段不再盲目承诺

  • 专一承诺Agent

    • 停下来确定意图是否已经实现了或者意图是否已经不可能实现
    • 问题：当前意图可能不再是最佳的。因为在规划和动作执行的过程中，环境可能发生变化

  • 坦率承诺Agent

    • 在每次执行一个动作后，都会有慎思过程来重新计算意图

  • 使用意图重考虑的Agent

    • 专一承诺Agent：不停下来充分地重新考虑它的意图，可能这些意图不再是最佳的
    • 坦率承诺Agent：不断地重复考虑它的意图，将会没有足够的时间实现它的意图
    • 解决：引入一个布尔函数 reconsider

  • PRS 过程推理系统

    • 采用 BDI 体系结构
    • 在 PRS 系统中，慎思过程是从不同的规划中选择的过程
    • Agent 如何慎思
      • 通过使用元级规划来实现，可以在执行时修改Agent的意图
      • 可以对规划使用效用，只需要选择效用最大的规划

2.5 反应式Agent

  • 代表性的工作：归类式体系结构
  • Brooks的主要观点
    • 智能行为无需明确使用符号AI建议的那类表示方法就能够产生
    • 智能行为无需使用符号AI建议的那类精确的抽象推理就能够产生
    • 智能是某种复杂系统自然产生的属性
  • 归类式体系结构的特点
    • Agent做决策，通过一个完成任务的行为集合实现
    • 很多行为可以同时“触发”
    • 通过一个完成任务的行为集合做决策

2.6 混合式Agent

  • 根据不同层次的实现型体系结构之间的相关性和组织方式的差异，分成两种集成方式
    • 水平层次结构：Touring机
      • 每一个软件层都与传感器输入和动作输出直接相连
      • 仲裁函数，来决定在给定时间内由哪一层来控制Agent
      • 优点：概念简洁；缺点：仲裁部件的交互复杂度高
      • Touring 机：包含三个动作的产生层，可以彼此通信
        • 三个控制层
          • 模型层：代表世界上各种各样的实体（包括Agent本身和其他Agent）
            • 可以预言Agent之间的冲突，并产生需要完成的新目标来解决这些冲突
            • 新目标下传到规划层，利用规划层来决定如何实现它们
          • 规划层：完成Agent的预动行为
            • 有一个规划库，为了实现目标，尝试在规划库中找到一个合适的规划，据此选择出要执行的动作
          • 反应层：用情景-动作规则集实现，能对环境中发生的改变提供迅速的反应
        • 这三个控制层嵌入到控制子系统中
          • 控制子系统能有效地决定这些层中哪一个应该控制Agent
          • 动作仲裁部件，按照控制规则集来实现
    • 垂直层次结构：InteRRap
      • 每个传感器输入和动作输出最多与一层相关
      • 两种方法在不同层次的实现型体系结构之间显式地交互数据和控制信息
        • 一路结构
        • 二路结构
      • 优点：层次间交互的复杂度低；缺点：不能容错，即任何一层的故障都可能会对Agent的性能产生严重影响
      • InteRRaP：与Touring机的主要差别——层次与环境间交互的方式
        • 三个层次的行为决策
          • 行为层：处理反应行为
          • 规划层：处理日常规划来完成Agent的目标
          • 协作层：处理社会交互作用

第三部分：通信与合作

3.1 相互理解的Agent

3.2 通信

  • 言语行为 特点
    • 传达某种交互的意图和内容
    • 影响其他Agent的内部状态以及相应的行为实施
  • 言语行为理论
    • 把通信建模成可以改变通信参与者的思维状态的动作
    • 言语行为理论三分说
      • 以言指事：言语行为的物理动作
      • 以言行事：言者传递给听者的意图
      • 以言成事：言语行为的结果

3.3 合作

  • 合同网（CNET）协议是通过任务共享实现有效合作的高级协议

  • 结果共享：黑板系统、订阅/通知模式

  • 合作分布式问题求解（CDPS）

    • 仁慈性、自利性、一致性、协调性

  • 任务共享和结果共享

    • CDPS的三个阶段：问题分解、子问题求解、解综合
    • 任务共享
      • 一个任务的各个部分被分配给不同的Agent
      • 例子：合同网
        • 识别、通知、竞标、授予、实现
    • 结果共享
      • 信息（如部分结果）被分配
      • 例子
        • 黑板共享：通过共享的数据结构
        • 订阅/通知模式：一个对象订阅另一个对象，信息以预动的方式在对象间共享

  • 不一致性

    • 两个方面产生不一致性
      • 信念
      • 目标/意图
    • 产生不一致信念的原因
      • Agent不能看到全局信息
      • Agent拥有的传感器/访问的信息源有毛病
    • 产生不一致目标的原因
      • Agent是自治的，没有共享的目标
    • 处理不一致性的方法
      • 不允许出现
      • 通过协商解决不一致性
      • 建立一个不一致性出现后能平稳退化的系统

  • 协调

    • 协调问题：如何管理Agent活动之间的内部依赖关系
    • 如果Agent参加的活动中存在任何形式的相互作用，那么一定的协调机制是必需的
    • 部分全局规划、联合意图、互相建模、规范和社会法律

4.2 制定群组策略

社会选择理论

  • 每个投票者对候选者都有偏好：对集合 Ω\OmegaΩ 内元素的一个排序。将候选集合 Ω\OmegaΩ 上的所有偏好构成的集合记作 ∏(Ω)\prod(\Omega)∏(Ω)

  • 社会选择理论研究的基本问题：偏好聚合

  • 偏好聚合

    • 社会福利函数：输入是每个投票者的偏好，输出是一个社会偏好排序
    • 社会选择函数：只需选出一个候选者，而不是一个社会偏好排序

投票过程

  • 多数制

    • 最高票者当选，是一个社会选择函数，单一结果
    • 问题：可能多数人偏向其他候选者

  • 策略性投票

    • 通过策略性投票，获得更期望看到的结果

  • 康多塞悖论：在某些情况下，无论我们选择哪种结果，大多数选民都会对我们选择的结果不满意

  • 序列多数选举

    • 多数制的一种变体，表现为进行一系列的选举

      • 每次进行成对选举，胜者进入下一次成对选举

    • 序列可以是线性的、树状的

  • 多数图

    • 完全的、不对称的、不自反的
    • 如果存在议程使得某候选者为最终赢家，则称该候选者为一个可能的赢家
    • 康多塞赢家：对任意议程，该候选者都是最终赢家。即在任何成对选举中都能获胜的候选者

  • 波达计数

    • 多数制有较多问题的原因——忽略了大多数选民的偏好顺序，只关注排名靠前的候选人
    • 考虑了整个偏好顺序
      • 对于每个候选人，都有一个计数变量，计算支持这个候选人的强弱程度

  • 斯莱特排序

    • NP-难问题：找到一个一致排序（不包含环），使它尽可能接近多数图。（拓扑？）

投票过程的性质

  • 一个“好的”投票，三个关键性质
    • 帕累托条件：如果每个投票者都将 wiw_iwi​ 排在 wjw_jwj​ 前面，那么一定有 wi≻∗wjw_i\succ^* w_jwi​≻∗wj​
      • 多数制和波达计数满足帕累托条件，而序列多数选举不满足
    • 康多塞赢家条件：如果 wiw_iwi​ 是一个康多塞赢家，那么 wiw_iwi​ 在社会选择中排名第一，或者最终选择的是 wiw_iwi​
      • 只有序列多数选举满足
      • 康多塞赢家：在任何选举议程下都是最终赢家。即在任何成对选举中都能获胜的候选者（可用多数图判断是否是康多塞赢家）
    • 无关选项独立性：两个候选项的社会排序仅依赖于二者在投票者偏好中的相对关系，与其他候选项无关
      • 多数制、波达计数和序列多数选举都不满足IIA
  • 独裁性：如果对某个投票者 iii 有：f(w‾1,...,w‾n)=w‾if(\overline{w}_1,...,\overline{w}_n)=\overline{w}_if(w1​,...,wn​)=wi​，那么该投票过程具有独裁性
    • 具有独裁性的过程满足帕累托条件和IIA这两个好的性质
  • 阿罗定理
    • 对于两个以上候选人的选举，唯一满足帕累托条件和IIA的投票程序是独裁。其社会结果实际上是简单地由一个选民选择

4.3 形成联盟

合作博弈概览

  • 合作博弈建模 Agent 能通过合作获利的情景
    • 特点1：使用合同等与其他 Agent 制定具有约束力的协议
    • 特点2：收益不直接分配给个体，而是集体
  • 合作博弈的三个阶段
    • 联盟结构的生成
    • 求解每个联盟的最优化问题
    • 划分利益
  • 合作博弈：G=<Ag,v>\mathcal{G}=<Ag,v>G=<Ag,v>
    • AgAgAg 是 Agent 的集合
    • v:2Ag→Rv:2^{Ag}\rightarrow \mathbb{R}v:2Ag→R：合作博弈的特征函数，表示一个联盟可以获得的收益
  • 一个 Agent 应该只加入满足下列条件的联盟 CCC
    • 可行的：不反对联盟 CCC 的收益向量
    • 高效的：联盟 CCC 的所有收益都得到了分配（v(C)=∑i∈Cxiv(C)=\sum_{i\in C}x_iv(C)=∑i∈C​xi​）（收益向量元素之和）
  • 合作博弈的核心：对所有成员（大联盟）的收益的可行分配集合，使得没有任何子联盟反对

收益公平分配：夏普利值

  • 夏普利值：定义如何公平分配合作收益

  • Agent i 的夏普利值：在各种可能的联盟次序下，Agent i 对联盟的边际贡献的平均值

4.4 分配稀缺资源

拍卖及其分类

  • 商品的价值

      • 对拍卖中的每个买方而言，商品的公共价值相同
    • 收藏价值
      • 只与 Agent 的主观判断有关，而与其他 Agent 都无关
    • 相对/相关价值

4.6 辩论

  • 辩论的方法

    • 抽象辩论：论证具有原子性，没有结构，是不可分割的实体
    • 演绎辩论：具有逻辑结构，能通过逻辑演绎进行推理

  • Dung式抽象辩论系统 <Σ,⊳><\Sigma,\rhd><Σ,⊳> 表示 Dung 式抽象辩论系统

    • 无冲突的立场
      • 如果立场 S 中没有论证攻击 S 中的其他论证，那么 S 是无冲突的
    • 互相辩护的立场
      • 如果 S 的每一个被攻击的论证被 S 中的一些论证辩护，那么 S 是互相辩护的（可自己辩护自己）
    • 可采纳的立场
      • 如果一个立场是无冲突并且互相辩护的，那么这个立场是可采纳的

  • 偏好拓展

    • 最大的可采纳集合
    • 加入另一个论证会使得这个立场变得不可采纳

  • 轻信的和怀疑的接受

    • 如果一个论证是每一个偏好拓展的元素，那么这个论证是被怀疑接受的
    • 如果一个论证是至少一个偏好拓展的元素，那么这个论证是被轻信接受的

  • 理性拓展

    • 理性拓展是最不可能被质疑的论证的集合
    • 只接受无法避免接受的论证
    • 只拒绝无法避免拒绝的论证

    • 目标是找到随机博弈的纳什均衡策略
    • 优点
      • 能保证每个Agent性能的下界

    • 独立学习算法、联合动作学习算法、WoLF-PHC算法
